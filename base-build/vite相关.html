# vite 相关总结

vite 是基于 rollup 的现代前端构建工具，它利用浏览器原生 ES 模块支持，实现开发时的快速冷启动、模块热更新，以及生产环境的代码压缩和优化。
vite 在开发环境下更快，直接启动服务器，按需编译，当浏览器请求某个模块时，才会编译相关文件和依赖。

# vite.config.js 配置

  • base：设置静态资源路径。
  • resolve：配置路径别名。
  • plugins：插件配置。
  • server：开发服务器配置。
  • build：构建配置，如代码分割、压缩等。

通过这些配置可以满足大部分项目的需求。优化策略包括代码分割、资源压缩、缓存优化、减少外部依赖和使用环境变量等，能够显著提升项目的性能和开发体验

// vite.config.js 基本配置示例
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import { resolve } from 'path';
import viteCompression from 'vite-plugin-compression';

export default defineConfig({
  base: './',
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'), // 将 @ 映射到 src 目录
    },
  },
  plugins: [
    vue(),
    viteCompression({ algorithm: 'gzip' }), // 启用 Gzip 压缩
  ],
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      },
    },
  },
  build: {
    // 通过 rollupOptions 可以自定义 Rollup 的打包行为
    rollupOptions: {
      output: {
        chunkFileNames: 'js/[name]-[hash].js', // 分块文件名
        assetFileNames: '[ext]/[name]-[hash].[ext]', // 静态资源文件名
      },
    },
  },
});

# 环境变量

  • .env：默认环境变量文件。
  • .env.development：开发环境变量文件。
  • .env.production：生产环境变量文件。
  • .env.[mode]：自定义环境变量文件。
// .env.development 通过 VITE_ 前缀来定义环境变量
VITE_API_URL=http://localhost:8080

// main.js
import { createApp } from 'vue';
import App from './App.vue';

const app = createApp(App);
// 通过 import.mate.env 来访问环境变量
app.config.globalProperties.$apiUrl = import.meta.env.VITE_API_URL;

app.mount('#app');
