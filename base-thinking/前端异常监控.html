# 前端异常监控，如何捕获异常，如何上报异常？

# 前端异常监控

前端异常监控，顾名思义就是监控前端代码的异常情况，及时发现问题，及时修复。前端异常监控主要包括以下几类：

  1. JavaScript 代码异常：包括语法错误、运行时错误等。
  2. 资源加载异常：包括图片、样式表、脚本等资源加载失败。
  3. 网络请求异常：包括 AJAX 请求、Fetch 请求等。
  4. 用户行为异常：包括点击、滚动、输入等用户行为异常。

# 如何捕获异常

  1. try...catch：使用 try...catch 可以捕获 JavaScript 代码中的异常，但是只能捕获同步代码中的异常，对于异步代码中的异常无法捕获。
  2. window.onerror：window.onerror 可以捕获 JavaScript 代码中的异常，包括同步和异步代码中的异常。但是，对于资源加载异常和网络请求异常无法捕获。
  3. window.addEventListener('error', callback)：window.addEventListener('error', callback) 可以捕获资源加载异常和网络请求异常，但是无法捕获 JavaScript 代码中的异常。
  4. window.addEventListener('unhandledrejection', callback)：window.addEventListener('unhandledrejection', callback) 可以捕获 Promise 异常，但是无法捕获同步代码中的异常。

# 如何上报异常

  1. 发送 HTTP 请求：可以使用 XMLHttpRequest 或者 Fetch API 发送 HTTP 请求，将异常信息发送到服务器。
  2. 发送 img请求：可以使用 img 标签发送 img 请求，将异常信息发送到服务器。

# 代码示例

// 捕获 JavaScript 代码异常
window.onerror = function (message, source, lineno, colno, error) {
  console.log('JavaScript 代码异常', message, source, lineno, colno, error);
};

// 捕获资源加载异常
window.addEventListener('error', function (event) {
  console.log('资源加载异常', event);
});

// 捕获 Promise 异常
window.addEventListener('unhandledrejection', function (event) {
  console.log('Promise 异常', event);
});

# 注意事项

  1. 性能问题：异常捕获和上报会带来一定的性能开销，需要权衡异常捕获的粒度和性能之间的平衡。
  2. 隐私问题：异常信息中可能包含敏感信息，需要确保异常信息的安全性和隐私性。
  3. 异常处理：对于捕获到的异常，需要根据具体的业务场景进行相应的处理，比如记录日志、发送报警等。

# 如何设计一个前端异常监控系统，例如 Sentry。

# 功能需求

  1. 异常捕获：捕获 JavaScript 代码异常、资源加载异常和 Promise 异常。
  2. 异常上报：将捕获到的异常信息发送到服务器。
  3. 异常分析：对上报的异常信息进行分析和处理，比如统计异常发生的频率、异常的堆栈信息等。
  4. 异常处理：根据具体的业务场景进行异常处理，比如记录日志、发送报警等。
