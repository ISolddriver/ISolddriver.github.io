# 浏览器的事件循环？

Javascript 是单线程的，浏览器的事件循环机制，是处理异步任务的一种机制。

# 在Javascript中，任务分为同步任务和异步任务。

  • 同步任务：在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行下一个任务。
  • 异步任务：不进入主线程、而进入"任务队列"（task queue）的任务，只有"任务队列"通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。

# 宏任务与微任务

  • 宏任务：script（同步代码）、setTimeout、setInterval、setImmediate、I/O、UI rendering。
  • 微任务：Promise.then、process.nextTick（Node.js）、MutationObserver。

# 事件循环机制

  1. 执行全局Script同步代码，这属于宏任务。
  2. 执行过程中，同步代码执行完毕，执行栈为空，查询是否有微任务需要执行，有则执行微任务。
  3. 当前宏任务执行完毕，查询是否有微任务需要执行，有则执行微任务，微任务执行完毕，查询是否有微任务需要执行，没有则查询是否有宏任务需要执行，有则执行宏任务，如此往复。

# async 与 await

async 和 await 是 JavaScript 中用于简化异步编程的语法糖，基于 Promise 实现。它们让异步代码看起来更像同步代码，提高了代码的可读性和可维护性。

  • async函数返回一个Promise对象，可以使用then方法添加回调函数。
  • await表达式会暂停其后的异步代码执行，等待Promise完成，并返回Promise的结果。
  • async函数内部return语句返回的值，会成为then方法回调函数的参数。

# 事件循环示例

async function async1() {
    console.log('async1 start')
    await async2()
    console.log('async1 end')
}
async function async2() {
    console.log('async2')
}
console.log('script start')
setTimeout(function () {
    console.log('settimeout')
})
async1()
new Promise(function (resolve) {
    console.log('promise1')
    resolve()
}).then(function () {
    console.log('promise2')
})
console.log('script end')
// 输出结果:
// script start
// async1 start
// async2
// promise1
// script end
// async1 end
// promise2
// settimeout

# Generator函数

  1. 使用function*定义，通过yield暂停和恢复执行。
  2. 返回一个生成器对象，通过next() throw() return()方法控制执行。
  3. 可以实现迭代器，异步编程和惰性求值等功能。
function* myGenerator() {
    yield 1;
    yield 2;
    yield 3;
}
const gen = myGenerator();
console.log(gen.next().value); // 1
console.log(gen.next().value); // 2
console.log(gen.next().value); // 3
  • 调用Generator函数并不会立即执行函数体，而是返回一个生成器对象，通过next()方法逐步执行函数。
  • yield关键字用于暂停Generator函数执行，并返回一个值，每次调用next()方法时，Generator函数会从上次暂停的位置继续执行，直到遇到下一个yield关键字或函数结束。 -next()方法可以传入参数，该参数会作为yield表达式的返回值。

# async 与 await 实现原理，手写一个async/await

async 和 await是 JavaScript 中与用于处理异步变成的语法糖，本质上是基于Promise 和生成器Generator结合实现的。

1. 实现原理

function asyncGenerator(genFun) {
    return function (...args) {
        const gen = genFun(...args);
        function step(result) {
            if (result.done) return Promise.resolve(result.value)
            return Promise.resolve(result.value).then((res) => {
                return step(gen.next(res))
            }).catch((err) => {
                return step(gen.throw(err))
            })
        }
        return step(gen.next())
    }
}
const asyncFun = asyncGenerator(function* () {
    const a = yield Promise.resolve(1)
    const b = yield Promise.resolve(2)
    return a + b
})
asyncFun().then((res) => {
    console.log(res) // 3
})
  • asyncGenerator函数接受一个生成器函数作为参数，并返回一个函数。
  • 返回的函数接受任意数量的参数，并将这些参数传递给生成器函数。
  • step函数用于逐步执行生成器函数，它接受一个result参数，该参数是生成器函数的执行结果。
  • step函数首先检查result的done属性，如果为true，则表示生成器函数已经执行完毕，直接返回Promise.resolve(result.value)。
  • 如果result的done属性为false，则表示生成器函数还没有执行完毕，需要继续执行。
