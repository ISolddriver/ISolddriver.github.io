# 关于网络协议

  1. TCP/IP协议
  • TCP/IP协议是互联网通信的基础协议，它定义了计算机之间如何进行通信的规则和标准。
  • 应用层
    • 功能：直接面相用户提供网络服务接口，定义数据格式和交互规则。
    • HTTP(网页传输)、FTP(文件传输)、SMTP(邮件发送)，DNS等；
  • 传输层
    • 功能：实现端到端的数据传输控制，确保数据可靠和高效交付。
    • TCP(传输控制协议，三次握手)、UDP(用户数据报协议)；
  • 网络层
    • 功能：负责跨网络的逻辑寻址与路由，将数据包从源主机传输到目标主机。
    • IP(互联网协议)、ICMP(互联网控制消息协议，网络状态监测)；
  • 网络接口层
    • 功能：管理物理介质上的数据传输
    • 以太网，WiFi等；

# TCP为什么需要三次握手和四次挥手？

  • 三次握手：指的是建立一个 TCP 链接时，客户端与服务器需要发送 3 个包，确保双方的发送和接收能力都正常。
  • 四次挥手：指的是终止一个 TCP 链接时，客户端与服务器需要发送 4 个包，确保双方都能正常关闭连接。

# TCP和UDP的区别

  • TCP是面向连接的，UDP是无连接的；
  • TCP是可靠传输，UDP是不可靠传输；
  • TCP是面向字节流的，UDP是面向数据报的；
  • TCP的头部开销较大，UDP的头部开销较小；
